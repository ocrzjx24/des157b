feasibility report

features

  • vertical scrolling timeline that maybe the user can drag too. i like the idea of smooth scrolling involved
  • each node click triggers a full-page visual shift (colors, font, texture) to match the era
  • smooth transitions / animations, especially between era visual shifts
  • openable "book" panel that contains my research and data and stuffs
  • consistent nav element

tech required

  • html/css + js - for pretty much everything
  • GSAP ScrollTrigger - fires animations as the user scrolls, which could be cool
  • Locomotive Scroll - smooth inertia-based scrolling library, for smooth scroll effect
  • Granim.js - or NEAT Gradients (if i can remove the watermark), for visuals
  • Motion - fka Framer Motion. might look into other animation libraries too, but will be nice for making smooth and fancy interactions
  • Figma - creation of graphics (maybe Photoshop too?)
  • APIs - still looking into them. Scheduling might help with framerate, certain other ones e.g. ViewTransitions might help with optimizing anims

skill assessment

  • html + css - comfy, no concerns
  • custom css props + javascript - should be fine; theme change concept reminds me of my own portal page
  • js libraries - with enough effort and reading doc, i should be good (i hope)

difficulties

  • distinct themes - making each era feel different yet accurate is going to prove difficult, and i'm currently working on the visuals still (i want to do all i can to ensure it is up to par with my own standards)
  • licensing - i doubt this will be a major issue but i might run into some issues with fonts or something
  • user flow / balance - i still have to solidify the scope of my piece to ensure that it's not too much work for me, nor is it too boring for the user. i think 5-6 eras should be good though (each decade from 1950 - 2000)
  • mobile view - self-explanatory somewhat
